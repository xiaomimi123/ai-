# 软件著作权申请材料 · 使用说明

本目录是为《行政事业单位内控评价合规审查智能体软件 V1.0》申请**中国版权保护中心（CPCC）软件著作权登记**自动生成的材料。

```
软著申请材料/
├── README-使用说明.md     ← 你正在看的文件
├── 源代码.pdf             ← 60 页，提交版权中心
├── 软件说明书.pdf         ← 62 页，提交版权中心
└── build/                 ← 中间产物，不用提交
    ├── gen_source_pdf.py  ← 源代码 PDF 生成脚本
    ├── gen_manual_pdf.py  ← 说明书 PDF 生成脚本
    ├── 源代码.html
    ├── 软件说明书.html
    └── manual_src.md
```

---

## 一、提交前必改的占位字段

打开 `build/gen_manual_pdf.py`，改顶部常量后**重跑脚本**：

| 变量名 | 当前值 | 应该改成 |
|---|---|---|
| `APPLICANT` | `【请填写申请人 / 单位名称】` | 你的姓名 / 公司全称 |
| `SOFT_NAME` | 行政事业单位内控评价合规审查智能体软件 | 如有别名可调 |
| `SOFT_VERSION` | V1.0 | 按你内部版本号 |
| `COMPLETE_DATE` | 2026-06-08 | 实际开发完成日 |
| `PUBLISH_DATE` | 2026-06-08 | 首次发表日 |

> **首次发表日** = 首次给第三方看到 / 部署上线日。如果还没正式对外发布，可以填申请日。

重跑命令：
```
cd build
python3 gen_source_pdf.py    # 改了软件名/版本/日期才需要重跑这个
python3 gen_manual_pdf.py    # 改了任何字段都要重跑
```

## 四、需要你额外注意的事

1. **盖章 / 签字**
   - 个人申请：申请表需要本人签字
   - 公司申请：申请表需盖**公章**（不是合同章、不是部门章）

2. **截图建议补充**
   生成的说明书里**没有界面截图**。审查员通过截图判断软件真实存在。
   建议你自己抓 5-8 张关键界面截图（登录页 / 任务列表 / 工作底稿 / 报告页 / 后台管理），插入到说明书的"使用手册"章节对应位置。
   做法：把截图保存到 `build/screenshots/`，在 `build/manual_src.md` 里插入 `![](screenshots/login.png)`，重跑 `gen_manual_pdf.py`。

3. **审查周期**
   - 普通申请：30-60 工作日
   - 加急申请：1-5 工作日（加钱）

4. **费用**
   - 个人：300 元
   - 单位：300 元
   - 加急（1 个工作日）：约 1500 元

5. **代码原创性**
   生成的 60 页 PDF 都是你本仓库 `compliance-agent/` 下的原创业务代码，**没有**混入 FastAPI / SQLAlchemy 等第三方库本体。可以放心提交。

6. **登记号**
   通过后会拿到「软著登字第 XXXXXXXX 号」证书。证书是**纸质**的，会快递到申请表填的地址。

## 五、如果要改/重生成

所有 PDF 都由 `build/` 下两个 Python 脚本生成。改完顶部配置（软件名 / 申请人 / 日期 / 添加的源码文件清单）重跑即可，不需要任何第三方库。

依赖的系统组件：
- Python 3（macOS 自带）
- Google Chrome（用于 headless 打 PDF）
- pandoc（说明书生成需要：`brew install pandoc`）

如果换台机器跑，确保 `gen_source_pdf.py` 顶部的 `chrome` 路径正确：
```
/Applications/Google Chrome.app/Contents/MacOS/Google Chrome
```
