# 内控评价合规审查智能体

> **AI 替你逐项核查被检查单位的内控评价材料**——把审计师 **数天的人工核查压缩到分钟级**，让你专注在最需要判断的环节。

---

## 一、行业痛点

行政事业单位每年要做一次内部控制评价并上报上级。审计师和复核人面对的现实是：

- **被检查单位提交几十至上百份佐证材料**（制度文件 / 会议纪要 / 财务报表 / 合同台账……）
- **要按 54 项评价指标**逐项判断每份材料是否合规、是否真实、是否对应正确指标
- **每条指标还要查法规依据**、扣分理由、写专业评语
- **人工核查一个单位耗时 2-3 个工作日**，重复劳动多、易遗漏、判断标准不一

## 二、产品定位

> 站在 **上级复核方** 视角，给审计师一把 AI 助手。

不是替代审计师，而是把**重复性的"看、对、抄"工作交给 AI**，让审计师专注在**真正需要专业判断的环节**——评分调整、修订意见、定稿签字。

## 四、核心能力

### 1. AI 阅读自动绑定材料

上传整个文件夹 → AI 读取每份材料内容（不是看文件名）→ 自动识别属于 54 项指标的哪一项。

**实测**：23 份未绑定材料，文件名是「德市图〔2026〕1 号 ...」这种编号，AI 阅读后 **23/23 全部正确绑定**，命中率 100%。

### 2. AI 阅卷打分

对每条指标按「核查要点 + 扣分规则」给出：
- **核查后得分**（0 到指标满分之间）
- **审计师评语**初稿（可改）
- **7 个判定维度**：真实可靠 / 完整 / 合规 / 跨单位重复 / 匹配度

### 3. 工作底稿在线编辑

每条指标一行，**单元格直接编辑**：
- 改「核查后得分」→ blur 自动保存
- 写「调整得分说明」→ 自动保存
- 点 chip 切换「材料真实可靠 / 涉嫌造假」等判定

### 4. 状态机与定稿

```
草稿（AI 生成）→ 复核中（人工编辑）→ 已定稿（锁定为只读）
```

定稿后底稿不可改，确保上报材料和系统数据完全一致。

### 5. Excel 底稿 1:1 复刻模板

底稿可一键下载为 Excel，**列顺序 / 表头 / 合并单元格 / 复选框格式**完全按官方核查工作底稿模板，可直接上交。

### 6. 跨任务材料查重

材料上传时自动计算指纹，与库内其他单位的材料对比。**A 单位用 B 单位的材料"抄送"将被标记**。

### 7. 5 大问题维度统一归类

所有 AI 检出的疑点严格归到 5 大维度：
- **真实性问题**（公章 / 签字 / 年度 / 正式性）
- **完整性问题**（要素 / 章节 / 材料缺失）
- **合规性问题**（制度 / 流程 / 法规违规）
- **重复性问题**（跨单位材料重复）
- **匹配性问题**（材料与指标不匹配）

### 8. 法规库 RAG

支持上传上位法 / 评价办法 / 编报指南等。AI 核查时自动检索相关条款，每条问题附带「依据法规第 X 条」。

### 9. 全程留痕可溯源

每次"改单元格 / 忽略疑点 / 定稿"都写入审计日志。后台管理可查"谁、什么时候、改了什么"。

### 10. 角色权限分级

- **超级管理员**：所有权限 + 解锁定稿底稿
- **审查员**：核查 / 复核 / 定稿
- **被检查单位**：仅能看自己单位的任务
- **只读用户**：只看不改

## 五、典型工作流

```
① 准备阶段（一次性）
   评价指标库 ← 上传 Excel/JSON 自动识别
   问题清单库 ← 上传 JSON
   法规库     ← 上传 PDF/Word，AI 自动分类 + 向量化
       ↓
② 新建任务
   选择被检查单位 + 评价年度 + 核查范围 + 快速 / 精确模式
       ↓
③ 上传材料（拖文件夹）
   AI 阅读内容 → 自动绑定到 54 项指标（命中率 91-100%）
   未自动绑定的可手动下拉指定
       ↓
④ 触发 AI 核查
   实时进度条："正在核查 第 17/54 项 I-17 预算预警整改"
   快速模式：5-10 分钟
   精确模式：10-20 分钟
       ↓
⑤ 工作底稿（AI 阅卷产物）
   54 行明细 + 5 对材料判定 + 5 维度疑点
       ↓
⑥ 在线编辑复核
   改"核查后得分" / 写"调整得分说明" / 切换材料判定
       ↓
⑦ 完成复核，定稿
   底稿锁定为只读
       ↓
⑧ 下载 Excel 底稿（1:1 复刻官方模板）
   直接交付上级
```

## 八、安全与合规

- 所有数据**存储在客户私有服务器**，AI 调用可配置使用国产大模型
- 用户密码 bcrypt 加密存储
- 操作日志全留痕，可追溯任意一条数据修改的"人 / 时间 / 内容"
- 角色级数据隔离（被检查单位用户只能看到自己单位的数据）
- 支持 HTTPS + 反向代理部署
